# HyperIDE

HyperIDE — это визуальный конструктор и редактор React-компонентов, обеспечивающий манипуляцию компонентами в реальном времени через интуитивный drag-and-drop интерфейс. Он предоставляет мощный опыт редактирования на основе канваса с живым предпросмотром, интроспекцией компонентов и бесшовной интеграцией с существующими React-проектами.

## Возможности

### Визуальное редактирование компонентов

- **Редактор канваса в реальном времени**: Визуальное редактирование React-компонентов с мгновенной обратной связью
- **Drag & Drop интерфейс**: Интуитивное размещение компонентов и управление иерархией
- **Древовидное представление компонентов**: Иерархическая визуализация структуры компонентов с интеграцией AST
- **Редактор свойств**: Динамическая модификация свойств с поддержкой TypeScript и валидацией

### Управление проектами

- **Поддержка нескольких проектов**: Одновременное управление несколькими React-проектами
- **Интеграция с Git**: Клонирование и работа с Git-репозиториями напрямую
- **Определение фреймворка**: Автоматическое определение и конфигурация для Vite, Next.js и других фреймворков
- **Изоляция через Docker**: Опциональная контейнеризация Docker для безопасного выполнения проектов

### Система компонентов

- **Реестр компонентов**: Динамическая регистрация и управление React-компонентами
- **HTML-компоненты**: Встроенная поддержка HTML-элементов с полным контролем атрибутов
- **Пользовательские компоненты**: Импорт и использование компонентов из внешних проектов
- **Генерация пропсов**: Генерация дефолтных пропсов для компонентов с помощью AI

### Продвинутые возможности

- **Изоляция через iframe**: Безопасный рендеринг компонентов в изолированных iframe-контекстах
- **Манипуляция AST**: Прямая манипуляция абстрактным синтаксическим деревом для генерации кода
- **Поддержка Tailwind CSS**: Встроенный парсинг и применение Tailwind-утилит
- **Система отмены/повтора**: Полное управление историей для всех операций на канвасе
- **Интеграция с AI**: Интеграция с OpenAI/Claude/GLM для генерации компонентов

## Основные компоненты

### Canvas Engine

Canvas Engine (`client/lib/canvas-engine/`) — это сердце системы визуального редактирования:

```typescript
class CanvasEngine {
  // Регистрация и управление компонентами
  registerComponent(definition: ComponentDefinition)
  unregisterComponent(type: string)

  // Манипуляция экземплярами
  loadInstances(componentType: string, props: any, parentId: string | null)
  updateInstance(id: string, updates: Partial<CanvasNode>)
  deleteInstance(id: string)

  // Операции с деревом
  moveNode(nodeId: string, newParentId: string, index: number)
  duplicateNode(nodeId: string)

  // Управление историей
  undo()
  redo()

  // Сохранение состояния
  getSnapshot(): CanvasSnapshot
  loadSnapshot(snapshot: CanvasSnapshot)
}
```

**Ключевые проектные решения:**

- **Событийно-ориентированная архитектура**: Все изменения состояния генерируют события для обновления UI
- **Иммутабельные операции**: Каждое изменение создает новый снапшот состояния
- **Пакетный режим**: Несколько операций могут быть сгруппированы для единого обновления UI
- **Система плагинов**: Операции модульны и расширяемы

## Архитектурные решения

### 1. Рендеринг компонентов на основе iframe

**Проблема**: Необходимо рендерить пользовательские компоненты без загрязнения контекста основного приложения.

**Решение**: Использование iframe с same-origin прокси для изолированного рендеринга с сохранением доступа к DOM.

**Преимущества**:

- Полная изоляция стилей
- Независимые экземпляры React
- Граница безопасности для недоверенного кода
- Поддержка горячей замены модулей

**Компромиссы**:

- Накладные расходы на коммуникацию между фреймами
- Сложная обработка событий
- Соображения производительности для больших деревьев

### 2. Анализ компонентов на основе AST

**Проблема**: Необходимо понимать структуру компонентов без выполнения кода.

**Решение**: Парсинг компонентов с использованием Babel AST и статическое извлечение структуры.

**Преимущества**:

- Безопасный анализ недоверенного кода
- Точное извлечение дерева компонентов
- Поддержка TypeScript и JSX
- Возможность генерации кода

**Компромиссы**:

- Ограничено статическим анализом
- Сложная логика парсинга
- Требует обработки source map

### 3. Событийно-ориентированное управление состоянием

**Проблема**: Несколько UI компонентов должны реагировать на изменения канваса.

**Решение**: Паттерн event emitter с централизованным управлением состоянием.

**Преимущества**:

- Слабосвязанные компоненты
- Предсказуемые обновления состояния
- Простое тестирование и отладка
- Поддержка отмены/повтора

**Компромиссы**:

- Сложность управления событиями
- Потенциальные утечки памяти
- Отладка косвенных обновлений

### 4. SQLite для управления проектами

**Проблема**: Необходимо постоянное хранение конфигурации проектов без внешних зависимостей.

**Решение**: Встроенная база данных SQLite с использованием sql.js.

**Преимущества**:

- Нулевая конфигурация
- ACID соответствие
- Поддержка сложных запросов
- Портативный файл базы данных

**Компромиссы**:

- Ограничение одного процесса
- Файловая производительность
- Совместимость браузера для sql.js

### 5. Опциональная изоляция через Docker

**Проблема**: Безопасное выполнение недоверенного кода проекта.

**Решение**: Опциональная контейнеризация Docker для выполнения проекта.

**Преимущества**:

- Полная изоляция процессов
- Лимиты ресурсов
- Изоляция сети
- Консистентная среда

**Компромиссы**:

- Зависимость от Docker
- Накладные расходы на управление контейнерами
- Платформенные ограничения

### 6. Интеграция с Tailwind CSS

**Проблема**: Необходима гибкая система стилизации для генерируемых компонентов.

**Решение**: Динамический парсинг и применение классов Tailwind.

**Преимущества**:

- Utility-first подход
- Консистентная дизайн-система
- Малый размер бандла
- Поддержка IDE

**Компромиссы**:

- Многословность имен классов
- Кривая обучения
- Обработка во время сборки

## Документация API

### Управление проектами

#### `GET /api/projects`

Список всех проектов.

**Ответ:**

```json
[{
  "id": "uuid",
  "name": "Название проекта",
  "path": "/абсолютный/путь",
  "framework": "vite",
  "status": "running",
  "port": 5173
}]
```

#### `POST /api/projects`

Создание нового проекта.

**Запрос:**

```json
{
  "name": "Название проекта",
  "path": "/путь/к/проекту",
  "framework": "vite",
  "devCommand": "npm run dev"
}
```

#### `POST /api/projects/clone`

Клонирование Git репозитория как проекта.

**Запрос:**

```json
{
  "repoUrl": "https://github.com/user/repo.git",
  "name": "Название проекта"
}
```

### Операции с компонентами

#### `GET /api/parse-component`

Парсинг AST структуры активного компонента.

**Ответ:**

```json
{
  "success": true,
  "componentName": "Button",
  "structure": [/* AST узлы */],
  "filePath": "/путь/к/компоненту.tsx"
}
```

#### `POST /api/inject-unique-ids`

Инъекция уникальных ID в элементы компонента.

**Запрос:**

```json
{
  "filePath": "/путь/к/компоненту.tsx",
  "idMap": {
    "0": "element-id-1",
    "1": "element-id-2"
  }
}
```

#### `POST /api/update-styles`

Обновление стилей элемента с помощью классов Tailwind.

**Запрос:**

```json
{
  "elementId": "element-id",
  "styles": {
    "className": "flex items-center p-4"
  }
}
```

### Интеграция с AI

#### `POST /api/generate-preview`

Генерация кода компонента с помощью AI.

**Запрос:**

```json
{
  "prompt": "Создай компонент кнопки",
  "framework": "react"
}
```

#### `GET /api/ai-config`

Получение конфигурации AI.

#### `PUT /api/ai-config`

Обновление конфигурации AI.

**Запрос:**

```json
{
  "provider": "claude",
  "apiKey": "sk-...",
  "model": "claude-3-sonnet"
}
```

## Схема базы данных

### Таблица Projects

```sql
CREATE TABLE projects (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  path TEXT NOT NULL UNIQUE,
  devCommand TEXT DEFAULT 'npm run dev',
  port INTEGER DEFAULT 5173,
  internalPort INTEGER DEFAULT 5173,
  framework TEXT DEFAULT 'vite',
  status TEXT DEFAULT 'stopped',
  atomComponentsPaths TEXT,        -- JSON массив
  compositeComponentsPaths TEXT,    -- JSON массив
  createdAt INTEGER NOT NULL,
  updatedAt INTEGER NOT NULL
);
```

### Таблица AI Config

```sql
CREATE TABLE ai_config (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  provider TEXT DEFAULT 'glm',
  apiKey TEXT NOT NULL,
  baseURL TEXT,
  model TEXT DEFAULT 'glm-4.6'
);
```

### Таблица Active Project

```sql
CREATE TABLE active_project (
  id INTEGER PRIMARY KEY CHECK (id = 1),
  projectId TEXT,
  FOREIGN KEY (projectId) REFERENCES projects(id)
);
```

## Разработка

### Предварительные требования

- Bun 1.2+
- Docker (опционально, для поддержки контейнеров)
- Git

### Установка

```bash
# Клонирование репозитория
git clone https://github.com/yourusername/hyper-canvas.git
cd hyper-canvas

# Установка зависимостей
bun install

# Создание файла окружения
cp .env.example .env

# Запуск сервера разработки
bun dev
```

### Команды разработки

```bash
# Запуск сервера разработки
bun dev

# Сборка для продакшена
bun run build

# Запуск тестов
bun test

# Проверка типов
bun run typecheck

# Форматирование кода
bun run format.fix

# Запуск продакшен сервера
bun start
```

### Конфигурация проекта

Система автоматически определяет и настраивает проекты на основе их структуры:

1. **Vite проекты**: Определяются по `vite.config.ts/js`
2. **Next.js проекты**: Определяются по `next.config.js`
3. **Create React App**: Определяются по `react-scripts` в package.json

### Переменные окружения

```bash
# API ключи (опционально)
ANTHROPIC_API_KEY=your_api_key

# Конфигурация сервера
PORT=8080
NODE_ENV=development

# Путь к базе данных (опционально)
DB_PATH=./data/hyper-canvas.db
```

## Участие в разработке

1. Форкните репозиторий
2. Создайте ветку для фичи (`git checkout -b feature/amazing-feature`)
3. Внесите изменения
4. Запустите тесты (`bun test`)
5. Коммит с использованием conventional commits (`feat: добавить потрясающую фичу`)
6. Запушьте в свой форк
7. Откройте Pull Request

### Стиль кода

- Используйте TypeScript для всего нового кода
- Следуйте существующим паттернам и конвенциям
- Используйте Prettier для форматирования
- Пишите тесты для новых фич

- Держите компоненты маленькими и сфокусированными
