# FAQ

Frequently asked questions about HyperIDE.

## General

### What is HyperIDE?

HyperIDE is a visual React component editor with AI assistance. It lets you edit React components like Figma while AI generates production-ready code.

### What frameworks are supported?

- Vite (React)
- Next.js (App Router & Pages Router)
- Remix
- Create React App
- Bun-based projects

### Is it free?

HyperIDE is open source. Check the [GitHub repository](https://github.com/hyperide/hypercanvas) for license details.

### Does it work offline?

The editor works offline, but:
- AI features require internet
- Docker must be running
- Git operations need connectivity

## Projects

### Can I use my existing project?

Yes! Clone your project from Git and HyperIDE will analyze and configure it automatically.

### Where are projects stored?

Projects are cloned to `cloned-projects/` in the HyperIDE directory. Original repositories are not modified.

### Can I push changes to Git?

Yes, HyperIDE doesn't interfere with Git operations. Use terminal or your preferred Git client.

### How do I delete a project?

Click the trash icon on the project card. This removes:
- Database entry
- Docker container
- Cloned files

Your original repository is not affected.

## Editor

### Can I edit multiple components at once?

You can have multiple browser tabs open, but each shows one component at a time.

### How do I switch components?

Use the file tree in Code mode or the component selector in the left sidebar.

### Why can't I select certain elements?

Some elements may be:
- Generated by frameworks (Next.js wrappers)
- Part of third-party components
- Locked by the component structure

### Can I use custom fonts?

Yes, add fonts to your project's public directory and reference them in CSS.

## Styling

### Why don't my styles appear?

Common causes:
- Element not selected
- Wrong CSS state (hover vs default)
- Style being overridden
- Framework styles have higher specificity

### Can I use CSS-in-JS?

HyperIDE generates Tailwind classes. CSS-in-JS libraries work but aren't visually editable.

### Why is Grid not available?

CSS Grid is only supported with Tailwind CSS. Tamagui projects use flexbox.

### Can I edit media queries?

Not directly in the visual editor. Use Code mode for responsive styles.

## AI

### What AI model is used?

The AI backend is configurable. Check project settings for current configuration.

### Is my code sent to external servers?

AI features send code context to AI providers. See privacy policy for details.

### Can AI access my private APIs?

No, AI cannot make external requests. It only generates code based on your prompts.

### Why does AI sometimes generate wrong code?

AI can:
- Misunderstand complex requirements
- Generate outdated patterns
- Make logic errors

Always review AI-generated code.

## Docker

### Why does HyperIDE use Docker?

Docker provides:
- Isolation between projects
- Consistent environments
- Safe execution of user code
- Easy cleanup

### Can I use Podman instead?

Not currently. Docker Desktop or Docker Engine is required.

### How much disk space is needed?

- Base images: ~2GB
- Per project: varies by dependencies
- Clean up with `docker system prune`

### Can I use Docker Compose?

HyperIDE manages containers directly. Docker Compose isn't supported.

## Limitations

### What can't HyperIDE do?

- Edit backend code (API routes need Code mode)
- Complex state management (visual editing is limited)
- Database operations
- External API integrations

### Are there component size limits?

Very large components may:
- Slow down the editor
- Exceed AI context limits
- Cause preview performance issues

### Does it support TypeScript?

Yes, TypeScript is fully supported and recommended.
